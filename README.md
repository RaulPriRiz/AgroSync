# AgroSync

Hemos optado por un modelo mixto porque nos parece el punto intermedio más equilibrado entre compartir conocimiento y mantener la sostenibilidad económica del proyecto. Nos gusta la idea de que las cooperativas puedan beneficiarse de una base de software abierta que además puede mejorar con las aportaciones de otros desarrolladores, pero también queremos que el proyecto tenga una vía clara de ingresos y de mantenimiento a largo plazo.

Licencia elegida: Licencia **Apache 2.0**

Apache 2.0 me parece la más adecuada para el núcleo abierto del proyecto.

**¿Por qué la elijo?**
Porque permite que cualquiera use, modifique y distribuya el código, incluso en proyectos comerciales, pero sin obligarles a liberar sus propias modificaciones si no lo desean. Esto fomenta la adopción y colaboración, sin que yo pierda el control total sobre la versión comercial del software. Además, protege al autor frente a responsabilidades legales y conserva el reconocimiento de autoría.


**Derechos que doy a otros**
Con la Apache 2.0, otros pueden:

-Usar el software libremente, tanto en entornos personales como comerciales.


-Modificarlo, adaptarlo a sus necesidades y redistribuirlo.


-Incluir partes del código en otros proyectos.


-Acceder al código fuente y aprender de él.



**Limitaciones que impongo**

-Cualquiera que use mi código debe mantener mi aviso de copyright y la licencia original.


-No se puede utilizar mi nombre o el de mi empresa para promocionar versiones modificadas sin mi permiso.


-No se ofrece garantía: el uso es bajo la propia responsabilidad del usuario.


-Los módulos cerrados (análisis y exportaciones) no se liberan bajo esta licencia; siguen siendo propiedad exclusiva mía.
